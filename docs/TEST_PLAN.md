# MemoDoc - 테스트 계획 (TEST_PLAN)

---

## 가설

업무 메모를 입력하면 AI가 즉시 6가지 형태의 유용한 문서를 생성할 수 있다.
사람이 직접 정리하는 것보다 빠르고, 결과물의 품질이 충분히 실용적이다.

---

## 검증 방법

- 실제 업무/교육 메모 샘플로 6탭 전체 생성 후 품질 직접 확인
- Notion에 붙여 넣었을 때 별도 수정 없이 사용 가능한지 체크
- Mermaid 다이어그램이 문법 오류 없이 렌더링되는지 확인
- 메모 유형(회의/교육/아이디어) 선택에 따라 요약 형식이 올바르게 분기되는지 확인

---

## 테스트 시나리오 및 실행 결과

| 항목 | 입력 | 기대 결과 | 결과 |
|---|---|---|---|
| API 키 저장 | 유효한 `sk-or-` 키 | localStorage 저장, 새로고침 후 유지 | ✅ |
| 잘못된 API 키 형식 | `sk-or-` 아닌 값 | 오류 메시지, 앱 정상 동작 | ✅ |
| 구 Gemini 키 자동 삭제 | `AIza...` 키가 localStorage에 존재 | 자동 삭제 후 재입력 유도 | ✅ |
| 빈 입력 생성 | 메모 없이 버튼 클릭 | 경고 메시지 표시 | ✅ |
| 짧은 메모 차단 | 10자 미만 입력 | "너무 짧아요" 경고 메시지 | ✅ |
| 6탭 순차 생성 | 회의 메모 텍스트 | 탭별 문서 스트리밍 출력, 진행률 표시 | ✅ |
| 도식화 탭 렌더링 | 프로세스 포함 메모 | Mermaid 플로우차트 렌더링 | ✅ |
| 도식화 이모지/특수문자 | 복잡한 한글 메모 | 파싱 오류 없이 렌더링 | △ (프롬프트로 완화) |
| 오류 시 재시도 | rate limit 오류 발생 탭 | 🔄 재시도 버튼으로 단일 탭 재생성 | ✅ |
| 메모 유형 분기 - 회의 | 회의 선택 + 회의 메모 | 결정 사항 / 다음 할 일 섹션 출력 | ✅ |
| 메모 유형 분기 - 교육 | 교육 선택 + 학습 메모 | 핵심 개념 / 배운 점 / 적용 포인트 출력 | ✅ |
| 내보내기 | 생성된 탭 클릭 | .md 파일 다운로드 | ✅ |

## 성공 기준 달성 여부

| 기준 | 목표 | 실측 | 달성 |
|---|---|---|---|
| 첫 텍스트 출력 | 3초 이내 | 약 3~5초 (무료 모델 한계) | △ |
| 6탭 오류 없이 완료 | 100% | rate limit 시 재시도로 해결 | △ |
| 내보낸 .md Notion 호환 | 수정 없이 사용 가능 | 마크다운 구조 정상 렌더링 | ✅ |
| Mermaid 렌더링 성공률 | 80% 이상 | 프롬프트 제약 후 약 85% | ✅ |

## 발견된 이슈 및 조치

| 이슈 | 원인 | 조치 |
|---|---|---|
| 6탭 병렬 요청 시 rate limit | 무료 모델 분당 한도 초과 | 순차 처리 + 3초 딜레이로 변경 |
| Mermaid 파싱 오류 | 이모지·줄바꿈·괄호 포함 노드 | 프롬프트에 노드 작성 규칙 명시 |
| 교육 메모에 회의 형식 출력 | 소형 모델의 조건부 지시 미준수 | 유형 선택 UI로 프롬프트 강제 분기 |
| 속도 저하 (전체 약 30~40초) | 순차 처리 + 무료 모델 응답 지연 | Phase 2에서 유료 모델 전환 시 개선 예정 |

## 기대 효과

- 회의 직후 문서화 시간 10분 → 1분 이하
- 다이어그램 작성 부담 제거
- 메모가 팀 공유 자산으로 즉시 전환 가능
